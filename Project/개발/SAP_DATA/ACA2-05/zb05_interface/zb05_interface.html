*& SAP ABAP 인터페이스의 이점 및 적용

*동일한 메소드이지만 다른 기능:
*인터페이스에서 선언된 메소드는 다른 클래스에서 사용될 때 다른 기능을 가질 수 있습니다.
*이는 인터페이스에서 메소드를 구현할 수 없기 때문에 해당 메소드만 선언할 수 있기 때문입니다.
*클래스는 필요에 따라 해당 기능을 구현하거나 사용합니다.
*또한 프로그램을 더 읽기 쉽게 만드는 공통 기능을 그룹화합니다.

*다중 상속: 상속은 다중 상속을 지원합니다.
*즉, 두 개 이상의 클래스가 동일한 메서드 이름을 가지지만 기능은 다릅니다.
*클래스는 여러 인터페이스를 구현할 수 있고 인터페이스에서 선언된 메서드를 상속할 수 있지만
*클래스는 단일 상속만 지원합니다.

*다형성(Polymorphism): 다형성은 괴짜 단어인 폴리(Poly)에서 파생된 것으로 다수를 의미하고
*모르프(Morph)는 형태를 의미합니다. 상속의 주요 목적은 런타임 다형성을 달성하는 것입니다.
*상속은 인터페이스에서 선언된 단일 메서드가 다른 기능을 가진 다른 클래스에서 사용될 수 있으므로
*다형성에 대한 기반을 제공합니다.
